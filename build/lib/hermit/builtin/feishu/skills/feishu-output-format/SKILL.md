当你的输出包含结构化内容时，Hermit 会使用 **RichCardBuilder** 把它渲染成更完整的飞书卡片布局；普通简短回复仍然直接走纯文本消息。

飞书卡片仅支持 Markdown 子集，以下规则**必须严格遵守**。

---

## 输出边界

- 你只负责输出**正文文本/Markdown 文本**，不要输出整张卡片的 JSON。
- 不要输出 `msg_type`、`content`、`schema`、`body`、`elements` 这类 webhook / card payload 字段。
- 如果第一行是 `# 标题`，Hermit 会把它提取到卡片 Header，并自动补 subtitle、状态标签和分节布局。
- 当前适配器不会替你上传图片或文件，因此不要输出依赖 `image_key`、`file_key` 的原生卡片 JSON。

---

## 何时使用卡片格式

系统会自动检测你的输出是否含有 Markdown 标记：
- **纯文本输出** → 作为普通消息发送（不走卡片渲染，更轻量）
- **含 Markdown/扩展标签** → 渲染为消息卡片

**简短回复请直接输出纯文本，不要人为加 Markdown：**
```
好的，已收到你的需求，稍后给你结果。
```

**复杂内容才使用 Markdown：**
- 有列表、标题、代码块、表格
- 包含多个章节或结构化信息
- 需要彩色文字、高亮、分割线等视觉强调

---

## 卡片结构（仅用于复杂回复）

你的输出第一行若以 `# 标题` 开头，该标题会被自动提取为卡片顶部 Header，不出现在正文中。

- **结构化长回复**：第一行写 `# 主题`，正文用 `##` 分节
- 每次回复只能有**一个 `#` 标题**
- `##` 分节越清晰，RichCardBuilder 布局越自然

---

## 支持的 Markdown 语法

**文本样式**
- `**加粗**` `*斜体*` `~~删除线~~` `***~~混合~~***`
- `` `行内代码` ``

**代码块**（⚠️ 仅飞书客户端 7.6+ 支持）
```
​```js
// 代码内容
​```
```

**标题**（仅支持一级和二级）
- `# 一级标题` → 卡片蓝色 Header（自动提取，不在正文）
- `## 二级标题` → 正文内的章节标题
- `### h3 及以下` ❌ 不支持，渲染为带 `###` 前缀的普通文字

**列表**（暂不支持缩进，内容只支持文本和链接）
- `- 无序列表项`
- `1. 有序列表项`
- ❌ 嵌套缩进列表不可用（飞书会压平为同一级别）
- ⚠️ **列表项内只能写纯文本或链接**，不能用 `**加粗**` / `` `代码` `` / `*斜体*` 等格式

**属性 / 键值对列表**（间距最紧凑的写法）：

用**单换行**而非双换行分隔各属性项，飞书渲染器对 `\n\n` 会插入约 20px 段落间距：

```
**位置：** 怀柔区怀北镇河防口村
**规模：** 11 条雪道（1 条高级道、7 条中级道）
**亮点：**
- 近靠长城
- 雪质口碑极佳
```

⚠️ 如果你用双空行分隔属性，每一项之间会出现很大的空白。

**链接与图片**
- `[链接文字](https://url)` — 只支持 http/https
- `![图片描述](https://image.url)`
- 如果没有稳定可访问的图片 URL，优先改用普通链接，不要虚构图片地址

**分割线**
- `---` — 前面**必须**有一个空行，否则不渲染
- ⚠️ **不要**在 `# 标题` 的下一行紧接 `---`，否则它会被提取为卡片副标题的内容而非分割线

---

## 飞书扩展语法（非标准 Markdown）

**Hermit 图片标签**（由图片工具返回，用于插入已上传的飞书图片）
```md
<feishu_image key='img_v2_xxx'/>
```
- 这是 Hermit 自定义标签，不是原生 Markdown
- 仅在工具明确返回该标签时使用
- 请原样保留，最好单独占一行
- 不要手写或猜测 `key`

**彩色文字**（改变文字本身的颜色）
```
<font color='red'>文本内容</font>
```
支持色值：`red` `green` `blue` `grey` `violet` `orange` 等 color token

**彩色标签**（带背景色的徽标/tag 样式，比 font 更醒目）
```
<text_tag color='violet'>标签文字</text_tag>
```
支持色值与 `<font>` 相同：`red` `green` `blue` `grey` `violet` `orange` 等

**@提及**
```
<at id='all'></at>          → @所有人
<at id='{open_id}'></at>    → @指定成员
<at email='{email}'></at>   → 通过邮箱@成员
```

**高亮块**（仅灰色，用于注意事项）
```
<highlight>
这里是高亮显示的内容
可以多行
</highlight>
```

**辅助文本**（字号更小的说明性文字，渲染为卡片底部小号斜体文本）
```
<note>这是辅助说明</note>
```
> ⚠️ 飞书卡片 Schema 2.0 不原生支持 `<note>` 标签，Hermit 会自动把它转为 `*斜体小字*` 渲染在卡片底部。

**表格**（最多 10 列，单卡片最多 5 个表格）
```
<table columns={[{"title":"列1","dataIndex":"col1"},{"title":"列2","dataIndex":"col2"}]} data={[{"col1":"值A","col2":"值B"}]}/>
```
表格语法复杂，**仅在用户明确需要表格时使用**；简单数据优先用 `**键**：值` 或列表。

**分栏布局**（横向并排展示内容）
```
<row>
<col flex=1>左侧内容</col>
<col flex=2>右侧内容（宽度是左侧的 2 倍）</col>
</row>
```

**按钮**（可跳转链接或触发消息）
```
<button type="primary" action="navigate" url="https://example.com">
查看详情
</button>
```
type 可选：`primary`（蓝色）、`default`（灰色）、`danger`（红色）

---

## 不支持的格式

- `> 引用块` — 不渲染，改用 `**注意**：` 加粗标注
- `- [ ] 任务列表` — 不渲染，改用普通 `- 列表`
- `| Markdown 表格 |` — 不渲染，改用飞书扩展 `<table>` 或 `**键**：值`
- `### h3` 及以下 — 改用 `**加粗文本**` 替代子标题
- 嵌套列表缩进 — 改用平铺列表

---

## 长度限制

单条回复控制在 **25 KB** 以内。超长时请：
- 分段摘要，不完整输出所有内容
- 代码/日志截取关键部分并注明省略

---

## 格式示例

**结构化回复**：
```
# 本次发布摘要

## 新功能
- Expert 对话切换支持 revokeMsg 参数
- 新增 nativeOnSend 回调通知

## 修复
- 修复 isStatusConfigured 判断逻辑错误

---
<note>发布时间：2026-03-09 | 版本：v2.3.1</note>
```

**带警告的回复**：
```
## 注意事项

<highlight>
以下操作不可撤销，请提前备份数据。
</highlight>

操作步骤：
- 步骤 1：确认影响范围
- 步骤 2：执行迁移脚本
```

**简短对话**：
```
好的，已完成以下操作：
- 任务 A ✅
- 任务 B ✅

如需调整请告知。
```

**新闻 / 排行 / 有序列表**（信息密度最佳写法）：
```
# 今日要闻 Top 3（2025年7月31日）

1. **中共中央政治局会议：十四五收官，十五五规划启动**
   中共中央政治局7月30日召开会议，决定今年10月在北京召开二十届四中全会…

---

2. **俄堪察加半岛8.7级强震，日本多工厂停产、200万人疏散**
   俄罗斯堪察加半岛7月30日发生8.7级强震，引发太平洋多国海啸警报…

---

3. **加沙援助现场遭枪击，50人死亡逾600人伤**
   以色列军队7月30日在加沙北部向等待领取援助物资的平民开火…

<note>信息来源：新闻联播、新浪新闻、澎湃新闻，截至今日。</note>
```

关键点：
- 列表项标题用 `**加粗**` 突出
- `---` 分割线用于在视觉上分隔每条新闻
- 来源、日期等元信息放入 `<note>` 标签，渲染为底部小字
- 不要在 `# 标题` 后紧接 `---`（否则会成为副标题）
