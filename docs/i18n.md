# Hermit i18n 设计

Hermit 当前的 i18n 目标不是“把所有字符串一次性翻译”，而是先把用户可见文案做成可维护、可回退的 catalog。

## 当前覆盖范围

当前已经接入的主要范围：

- CLI 顶层 help 文案
- 菜单栏 companion 文案
- `plugin.toml` 中的插件描述与变量描述

明确不翻的内容：

- 命令名
- tool name
- schema 字段名
- hook 名
- slash command 名
- 直接影响模型行为的协议字段

## locale 解析顺序

当前解析顺序：

1. `HERMIT_LOCALE`
2. `LC_ALL`
3. `LC_MESSAGES`
4. `LANG`
5. 默认 `en-US`

实现位于 [`hermit/i18n.py`](../hermit/i18n.py)。

## 标准化规则

当前别名：

- `en` -> `en-US`
- `en-us` -> `en-US`
- `zh` -> `zh-CN`
- `zh-cn` -> `zh-CN`
- `zh-hans` -> `zh-CN`

如果是一般形态的 `lang-region`，会被标准化成：

- 语言小写
- 地区大写

例如：

- `fr-fr` -> `fr-FR`

## catalog 结构

当前文案目录：

- `hermit/locales/en-US.json`
- `hermit/locales/zh-CN.json`

格式使用扁平 key：

```json
{
  "cli.app.help": "Hermit personal AI agent CLI.",
  "menubar.title": "Hermit",
  "plugin.scheduler.description": "Scheduled task execution..."
}
```

不使用英文原句作为 key，避免未来改文案时把代码引用一起拖动。

## 代码使用方式

Python 层统一通过 `tr()` 取值：

```python
from hermit.i18n import tr

text = tr("cli.app.help", locale="zh-CN")
```

找不到 key 时：

- 优先回退到 `default=...`
- 再回退到 key 自身

这保证了缺 catalog 时不会阻塞程序启动。

## `plugin.toml` 的本地化方式

插件描述支持：

- `description_key`
- `description`

示例：

```toml
[plugin]
name = "usage"
description_key = "plugin.usage.description"
description = "显示当前会话的 token 消耗统计"
```

含义：

- 有本地化 catalog 时优先用 `description_key`
- 没有时回退到 `description`

变量描述同理。

## 当前边界

i18n 目前仍然只覆盖“人类可见文案层”，没有进入：

- agent system prompt 协议层
- tool schema 对模型的描述层
- Feishu 卡片模板与业务输出全面国际化

这是有意为之，目的是避免为了 UI 文案翻译而影响模型稳定性。

## 后续最值得继续接入的区域

下一批最有价值的接入点：

- CLI 子命令的 help 与错误提示
- Feishu 出站文案
- `ToolSpec.description` 中面向人类的说明部分
